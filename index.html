数据结构学习笔记

Posted on 2024-06-16

数据结构

第一章 基本概念和术语

1.数据 Data

外界信息的载体，如字符、文字、图象、声音等。

2.数据元素 Data Element 与数据项 Data Item

数据元素是数据的基本单位，可由若干个数据项组成，数据项为最小数据单位，不可分割。
数据项有时成为字段Field或域Domain。

3.数据对象 Data Object

是性质相同的数据元素的集合，是数据的一个子集。

4.数据类型 Data Type

是 高级程序设计语言 中的概念，是对数据取值范围和对数据进行操作的综合，数据类型规定了程序对象的特性。

例如 C# 语言中的字符串类型 String，一个 String 表示一个恒定不变的字符序列集合，所有的字符序列集合构成 String 的取值范围。我们可以对 String 进行求长度、复制、连接两个字符串等操作。

数据类型可分为两类：一类是非结构的原子类型，如 C#语言中的基本类型（整型、实型、字符型等）；另一类是结构类型，它的成分可以由多个结构类型组成，并可以分解。结构类型的成分可以是非结构的，也可以是结构的。

5.数据结构 Data Structure

数据结构是相互之间存在一种或多种特定关系的数据元素的集合。这种关系称为结构 Structure。

4种基本数据结构

  • 集合 Set
  • 线性结构 Linear Structure
  • 树形结构 Tree Structure
  • 图状结构 Graphic Structure

数据结构 (Data Structure) 简记为 DS
即 DS = (D,R)
D是数据元素的有限集合
R是数据元素之间关系的有限集合

数据结构包括数据的 逻辑结构 和 物理结构。

逻辑结构(Logic Structure)
数据的逻辑结构是从具体问题抽象出来的数学模型

存储结构(Storage Structure)
数据的物理结构，数据在计算机中的表示，又叫映像和存储

数据的存储结构包括顺序存储结构和链式存储结构两种。
顺序存储结构(Sequence Storage Structure)
链式存储结构(Linked Storage Structure)
链式存储结构中的数据元素称为结点(Node)，在结点中附设地址域(Address
Domain)来存储与该结点相邻的结点的地址来实现结点间的逻辑关系。这个地址
称为引用(Reference)，这个地址域称为引用域(Reference Domain)。

6.算法

6.1 算法定义与特性

算法(Algorithm)
是对某一特定类型的问题的求解步骤的一种描述，是指令的有限序列。
一个算法应该具备5个特点：

  • 有穷性 Finity
  • 确定性 Unambiguousness
  • 输入 Input
  • 输出 Output
  • 可行性 Realizability

6.2 算法的评价标准

  • 正确性
  • 可读性
  • 健壮性
  • 高效性

6.3 时间复杂度和空间复杂度

时间复杂度 Time Complexity
T(n) = O(f(n))

空间复杂度 Space Complexity
S(n) = O(f(n))

第二章 树的基本概念

1.树的定义

树(Tree)是n(n>0)个节点的有限集，当n=0时，称为空树
对于非空树
1.有且只有一个称为根的结点
2.除根节点外的结点可分为m(m>0)个互不相交的有限集，其中每一个集合本身也是一颗树，且称为树的子树(SubTree)

树的定义是递归的，即在树的定义中又用到了自身
树是一种递归的数据结构
树作为一种逻辑结构，同时也是一种分层结构，具有以下两个特点：
1.树的根结点没有前驱，除根结点外的所有结点有且只有一个前驱。
2.树中所有结点可以有零个或多个后继。

2.基本术语

1.结点：
树中的一个独立单元，包含一个数据元素及若干指向其子树的分支
2.结点的度：
结点所拥有的子树数目
3.树的度：
树中结点度的最大值
4.叶子：
没有子树的结点，也称为终端结点
5.非终端结点
有子树的结点
6.双亲和孩子
结点的子树称为结点的孩子，该结点称为子树的双亲
7.兄弟：
拥有相同双亲的结点
8.祖先
从根到该结点路径上所经过的所有结点
9.子孙
以该结点为根的所有结点
10.层次
从根算起，根为第一层，根的子树为第二层，根的子树的子树为第三层……
11.堂兄弟
双亲在同一层的结点互为堂兄弟
12.树的深度
最大层次数称为树的深度或高度
13.有序树
严格区分左子树和右子树的称为有序树，否则为无序树
14.森林：
多个不相交的树的集合

3.树的性质

1.树中的结点数为所有结点的度数+1
2.度为m的树中，第i层上至多有m^(i-1)个结点
3.高度为h的m叉树至多有(m^h-1)/(m-1)个结点
4.具有n个结点的m叉树的最小高度为logm(n(m-1)+1)

4.二叉树的定义

二叉树(Binary Tree)是n(n>=0)个结点所构成的集合
结点为零称为空树
对于非空二叉树T
1.有且只有一个根结点
2.除根结点外，其余结点分为互不相交的两个子集T1和T2，分别称为T的左子树和右子树
特点：
1.二叉树每个结点至多只有两个子树
2.二叉树的子树有左右区分

5.特殊二叉树

1.斜树

只有左子树或只有右子树的二叉树，又称为左斜树或右斜树

2.满二叉树

树的高度为h，且含有2^h-1个结点的二叉树，即每层都有最多的结点，子叶都在最深一层

3.完全二叉树

高度为h，有n个结点，当且仅当每个结点都与高度为h的满二叉树中编号为1~n的结点一一对应时
(1) 叶子结点只可能在层次最大的两层上出现。对于最大层次中的叶子结点，都依次排列在该层最左边的位置上
(2) 若有度为1的结点，则只可能有一个，且该结点只有左孩子而无右孩子(重要特征)
(3) 按层序编号后，一旦出现某结点(编号为i)为叶子结点或只有左孩子，则编号大于i的结点均为叶子结点。
(4) 若n为奇数，则每个分支结点都有左孩子和右孩子;若n nn为偶数，则编号最大的分支结点(编号为n/2)只有左孩子，没有右孩子，其余分支结点左、右孩子都有。

4.二叉排序树

左子树上所有结点的关键字均小于根结点的关键字，右子树上的所有结点的关键字都大于根结点的关键字
左右子树各又是一棵平衡二叉树

5.平衡二叉树
树上任一结点的左子树和右子树的深度之差不超过1

6.二叉树的性质

  1. 结点数量为n，则边的数量为n-1
  2. 非空二叉树的叶子结点等于度为2的结点数+1，即n0 = n2 + 1
  3. 非空二叉树上第k层至多有2^(k-1)个结点(k>0)
  4. 高度(或深度)为h的二叉树至多有2^h-1个结点(h>0)
  5. 具有n个结点的完全二叉树的高度为{log2(n)}+1
  6. 对完全二叉树按从上到下、从左到右的顺序依次编号1,2..∗,n,则有以下关系:
    (1)i>1时，结点i的双亲的编号为i/2，即当i为偶数时，它是双亲的左孩子；当i为奇数时，它是双亲的右孩子。
    (2)当2i≤n时，结点i的左孩子编号为2i, 否则无左孩子。
    (3)当2i+1≤n时，结点i的右孩子编号为2i+1，否则无右孩子。
    (4)结点i所在层次(深度)为{log2(i)}+1。
    7.深度为d(根深度为1)的二叉树只有度为0和2的结点，则此树至少有2d-1个结点

7.二叉树存储结构

1.顺序存储结构
n个结点的完全二叉树，数组存储，一层一层存储
非根结点的i父结点序号是i/2(取整)
结点i的左孩子是2i(若2i<=n，则没有左孩子)
结点i的右孩子是2i+1(若2i+1<=n，则没有右孩子)

2.链表存储
结点结构
leftchild–data–rightchild

8.二叉树的递归遍历

1.先序遍历
访问根结点->先序遍历左子树->先序遍历右子树

2.中序遍历
中序遍历左子树->访问根结点->中序遍历右子树

3.后序遍历
后序遍历左子树->后序遍历右子树->访问根结点

先序、中序和后序遍历过程中经过结点的路线相同，只是访问各结点的时机不同

9.二叉树的非递归遍历

非递归算法实现的基本思路：使用堆栈

1.中序遍历非递归
遇到结点，先压栈，并遍历左子树
左子树遍历结束，从栈顶弹出结点并访问
再按右指针中序遍历该结点右子树

第三章 堆的概念

堆的定义

优先队列：
特殊的”队列”，取出元素的顺序是依照元素的优先权（关键字）大小，而不是入队顺序

堆的操作

堆的主要操作有插入与删除

堆的实现

可以以数组、链表、有序数组和有序链表方式实现优先队列
使用完全二叉树实现

堆的特性

结构性：用数组表示的完全二叉树
有序性：任一结点的关键字是其子树所有结点的最大值（最小值）
最大堆(MaxHeap) 也称为大顶堆
最小堆(MinHeap) 也称为小顶堆

最大堆

抽象数据结构

类型名称：最大堆
数据对象集：完全二叉树，每个结点的元素值不小于其子节点的元素值
操作集：最大堆H，元素item

  • MaxHeap Create(int MaxSize)
  • Boolean IsFull(MaxHeap H)
  • Insert(MaxHeap H, ElemType item) 插入
  • Boolean IsEmpty(MaxHeap H)
  • ElemType DeleteMax(MaxHeap H) 删除

堆的结构

最大堆创建

最大堆的插入

在堆尾插入，与父结点比较大小，比父节点大则交换位置，直至交换到根结点

最大堆的删除

取出根结点（最大值）元素，同时删除堆的一个节点
将最后的一个元素放在根结点位置，比较左右子节点哪个更大，与其交换位置，直至没有子节点可比较，或子节点小于该元素

最大堆的建立

1.通过插入，将N个元素插入到初始为空的堆，时间复杂度为O(NlogN)
2.将N个元素按输入顺序存入，满足完全二叉树结构特性后调整结点位置，再满足最大堆的有序特性
时间复杂度为O(n)

第四章 哈夫曼树

哈夫曼树的定义

带权路径产长度(WPL)：设二叉树有n个叶结点，每个叶结点带有权值Wk，从根结点到每个叶结点长度为Lk，WPL即为长度乘对应权值之和

最优二叉树或哈夫曼树即为WPL为最小的二叉树

哈夫曼树的构造

每次将权值最小的两棵二叉树合并

哈夫曼树的特点

  • 没有度为1的结点
  • n个叶结点的哈夫曼树共有2n-1个结点
  • 有2个儿子的结点总数是叶结点总数-1，即n2=n0-1
  • 哈夫曼树的任意非叶结点的左右子树交换后仍是哈夫曼树
  • 同组权值的哈夫曼树可能不同构，但WPL相同

哈夫曼树编码

给定字符串，如何对字符进行编码，使得字符串的编码存储空间最少
不等长编码
避免二义性：任何一个字符的编码都不是另一个字符的前缀
实现：使所有字符都在二叉树的叶结点上
即构建一个哈夫曼树

第五章 集合

集合的表示

集合运算：交、并、补、差，判断一个元素是否属于某个集合
并查集：集合并、查某元素属于哪个集合

集合的存储结构

使用双亲表示法的树结构表示，树的根结点表示某个集合

数组存储形式
单一元素类结构类型

集合的查找

集合的并运算

分别找到x和y两个元素所在集合的根结点
如过根结点不同，则将其中一个根结点的父节点指针设置成另一个根结点的数组下标

集合的简化表示

因为集合中数据为正整数，因此集合中的N个元素都可以映射为整数0~N-1
所以可以使用数组下标作为元素的数据域，即1号机即为下标为0，数组中存储的值即为集合的根结点下标，或者上一个数据的下标
根结点存储的值为负数，可以以绝对值存储集合的大小
