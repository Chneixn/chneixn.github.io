Unity

Unity 的热更新

阅读时长: 5 分钟

热更新

HybridCLR + Addressable

环境确认

1.1 Unity

1.1.1 使用的Unity版本为2022.3.13f1

1.1.2 安装UnityComponent添加对IL2CPP的支持

笔记：https://blog.csdn.net/Chenxin_37/article/details/136747145

1.2 Visual Studio

1.2.1 安装“使用Unity的游戏开发”和“使用C++的游戏开发”套件

1.2.2 安装Window10-SDK最新版即可

框架搭建

2.1 创建新项目

2.2 安装HybridCLR

安装详看文档

文档主页：https://hybridclr.doc.code-philosophy.com/docs/beginner/quickstart

国内Git地址：https://gitee.com/focus-creative-games/hybridclr_unity.git

国外Git地址：https://github.com/focus-creative-games/hybridclr_unity.git

2.3 创建Asset/HotUpdate文件夹

2.4 在HotUpdate文件夹创建HotUpdate的程序集（Assembly Definition）

2.5 在HotUpdate文件夹任意创建一个脚本

2.6 在Visual Studio重载项目，此时的解决方案资源管理器应该显示两个程序集，如图

2.7 选择Unity的上方菜单项HybridCLR/Installer…

选择Install（安装成功后控制台会打印成功的信息，以及绿框位置显示为True）

2.8 选择Unity的上方菜单项HybridCLR/Settings…

2.9 在热更新Assembly Definition中添加在HotUpdate文件夹中的程序集

如果没有显示HybridCLR Settings，重启Unity

2.10 选择Unity的上方菜单项Edit/Project Settings…

在左侧选择Player

在下方展开Other Settings

2.11 按照图中选项修改

2.12 选择Unity的上方菜单项HybridCLR/Generate/All

2.13 将{proj}/HybridCLRData/HotUpdateDlls/StandaloneWindows64(MacOS下为StandaloneMacXxx)目录下的HotUpdate.dll重命名为HotUpdate.dll.bytes，将其复制到Asset/HotUpdate/Dlls内（无则新建）

（可选：对项目进行一次打包）

2.14 在HotUpdate文件夹下的所有脚本发生任何变更后，都需要在Unity的上方菜单项选择HybridCLR/CompileDll/ActiveBuildTarget

安装Adressable

3.1 打开Package Manager窗口

选择Package：Unity Registry

搜索Addressable

点击Install安装

3.2 选择Unity的上方菜单项Window/Asset Management/Addressable/Groups

选择Generate…

选择Profile：Default

选择Manage Profiles

3.3 修改设置

选择Remote 为 Custom

将Remote.LoadPath修改为自己的远程或本地的http资源服务器，路径内文件为本地的{project}/ServerData/[BuildTarget]

3.4 点击Addressables Groups窗口的New

创建一个新的组并命名为HotUpdateDll

选择组，在右侧Inspector窗口修改Build & Load Paths 为 Remote

3.5 在Project窗口找到AddressableAssetSettings

在Inspector窗口勾选Build Remote Catalog

3.6 选择Asset/HotUpdate/Dlls/HotUpdate.dll

（此时该文件在文件管理器中的后缀应为.bytes）

将其拖入Addressables Groups窗口的HotUpdateDll组内（打上标签Dll）

3.7 选择Play Mode Script里的第三项Use Exiting Build（Window）

3.8 点击Player Mode… 左侧的Build，选择New Build/Defalut Build Script

此时在{Project}/ServerData下会有文件夹为StandardWindow64文件夹

此文件夹内文件需与资源服务器内容同步

脚本热更新操作步骤

4.1 在HotUpdate文件夹下的所有脚本发生任何变更后，都需要在Unity的上方菜单项选择HybridCLR/CompileDll/ActiveBuildTarget

4.2 在Addressables Groups窗口选择Build/New Build/Defalut Build Script

4.3 同步ServerData文件夹内的文件至资源服务器

4.4 运行客户端下载

在HotUpdate文件下的脚本注意

5.1 在引用其他命名空间时，在客户端有可能会出现Script Missing错误

因此需要选择HotUpdate文件夹里的HotUpdate程序集

在Inspector勾选Allow ‘unsafe’ Code

在Assembly Definition Reference添加引用的命名空间程序集

5.2 在Assets/StreamingAssets文件夹放置对应所需的Dll文件

并在客户端的资源检查脚本第一时间读取
